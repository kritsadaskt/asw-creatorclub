CIS — Customer Information System

Website Lead Registration API  ·  Real Estate Integration

v15.0
Overview

This API allows external websites and partner systems to register real-estate leads directly into the CIS (Customer Information System). It provides master-data lookup endpoints and two lead-submission endpoints — one for shop/partner channels and one for website-originated contacts.

Test (UAT) http://aswinno.assetwise.co.th/CISUAT Production https://api.assetwise.co.th Production HTTPS https://api.assetwise.co.th/cis Base Path {Url}/api/{Module}/{EndpointName} Modules Master   Customer
Authentication

All requests must include an HTTP Basic Authentication header. The credentials are Base64-encoded and passed as a Bearer token.

Header:  Authorization: Basic <base64-credentials>
Master Endpoints (GetProjects — legacy token):
Authorization: Basic Y3VzdG9tZXJtYW5hZ2VtZW50OmN1c3RvbWVybWFuYWdlbWVudEAyMDE4
All Other Endpoints:
Authorization: Basic YXN3X2Npc19jdXN0b21lcjphc3dfY2lzX2N1c3RvbWVyQDIwMjMh
Contact the CIS integration team to obtain the correct credentials for each environment.
Standard Response Envelope

Every response is a JSON object with the following top-level structure:

{ "Success": true | false, "Message": "Save success." | "{error description}", "Data": {} | [] | [ {...}, ... ] }
FieldTypeDescription
Successbooleantrue on success, false on failure
MessagestringHuman-readable result or error description
Dataobject | arrayPayload for GET responses; empty {} or [] for POST responses

POST {Url}/api/Customer/SaveOtherSource SaveOtherSource

The primary endpoint for registering individual leads captured from a real-estate website. Supports UTM tracking parameters, appointment scheduling, consent flags, and an optional RegisterData wrapper that overrides all individual fields when present.

Fixed values for website integration:
 • ContactChannelID = 21 → Web Site
 • ContactTypeID = 35 → Register
 • FollowUpID = 42 → Yes

Integration Notes

Typical website registration flow:

  1. On page load, call GetProjects to get the correct ProjectID for the landing page.
  2. When the visitor submits the registration form, call SaveOtherSource with:
    • The resolved ProjectID
    • ContactChannelID = 21 (Web Site — fixed)
    • ContactTypeID = 35 (Register — fixed)
    • FollowUpID = 42 (Yes — fixed)
    • All UTM parameters captured from the URL
    • Consent flags from the form checkboxes
  3. Handle the Success: false case by showing an appropriate error to the user or logging for retry.
  4. Use GetContactChannels, GetContactTypes, and GetFollowUps only when building admin UIs or when the fixed IDs need to be validated.
Date / Time formats: Use YYYY-MM-DD HH:mm:ss for RefDate and YYYY-MM-DD for AppointDate. Use HH:mm for appointment time fields.
NULL values: For required string fields such as Tel and Email where data may genuinely be absent, pass the string "NULL" rather than omitting the field or sending a JavaScript null.
